The Difference between Classes and IDs

An ID works exactly like a class... Everything that I've written at the top applies to IDs as well. So what’s the difference?

An ID is identified by a # and a class is identified by a .

The barcode has information about this product eg: how much it cost and maybe which type of Ipod it is etc. All the Ipods of the same type/year will share the same barcode as they are gonna be priced the same $$.
This will be a class.

This Ipod will have as well a specific serial number that I like to call 'system ID'. This sysID will be unique to it as there's not gonna be a similar one in the world. This will be the ID.


Using Classes and IDs Semantically

It's important to have simple, useful and clear names when declaring this type of classes or IDs.

We can ask ourselves a few questions to get this right:

  • What's the function/purpose of the element that I'm gonna be assigned the class/ID to?


  • Is it the only element of it's type on the webpage? EG. How many menus does your page has? Maybe a left menu and a top menu?


Answering the first question: Let's say that the purpose of the element that we are tagging is to be a search input field on the page. We can use something like 'searchfield'.

Answering the second question: It's possible that you will have more than 1 element of that type on the page, so it’s useful to call this particular element something besides just 'searchfield'.

You could declare the class/id based on where this searchfield its possitionated on the page... Could be on the top menu and we could use something like 'topsearchfield' or 'leftsearchfield'.
